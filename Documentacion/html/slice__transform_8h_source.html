9 // Class for specifying user-defined functions which perform a
10 // transformation on a slice. It is not required that every slice
11 // belong to the domain and/or range of a function. Subclasses should
12 // define InDomain and InRange to determine which slices are in either
13 // of these sets respectively.

40  // Transform(s)=Transform(`prefix`) for any s with `prefix` as a prefix.
41  //
42  // This function is not used by RocksDB, but for users. If users pass
43  // Options by string to RocksDB, they might not know what prefix extractor
44  // they are using. This function is to help users can determine:
45  // if they want to iterate all keys prefixing `prefix`, whetherit is
46  // safe to use prefix bloom filter and seek to key `prefix`.
47  // If this function returns true, this means a user can Seek() to a prefix
48  // using the bloom filter. Otherwise, user needs to skip the bloom filter
49  // by setting ReadOptions.total_order_seek = true.
50  //
51  // Here is an example: Suppose we implement a slice transform that returns
52  // the first part of the string after spliting it using deimiter ",":
53  // 1. SameResultWhenAppended("abc,") should return true. If aplying prefix
54  // bloom filter using it, all slices matching "abc:.*" will be extracted
55  // to "abc,", so any SST file or memtable containing any of those key
56  // will not be filtered out.
57  // 2. SameResultWhenAppended("abc") should return false. A user will not
58  // guaranteed to see all the keys matching "abc.*" if a user seek to "abc"
59  // against a DB with the same setting. If one SST file only contains
60  // "abcd,e", the file can be filtered out and the key will be invisible.
61  //
62  // i.e., an implementation always returning false is safe.
63  virtual bool SameResultWhenAppended(const Slice& prefix) const {
64  return false;
65  }
66 };
