550  // Compact the underlying storage for the key range [*begin,*end].
551  // The actual compaction interval might be superset of [*begin, *end].
552  // In particular, deleted and overwritten versions are discarded,
553  // and the data is rearranged to reduce the cost of operations
554  // needed to access the data. This operation should typically only
555  // be invoked by users who understand the underlying implementation.
556  //
557  // begin==nullptr is treated as a key before all keys in the database.
558  // end==nullptr is treated as a key after all keys in the database.
559  // Therefore the following call will compact the entire database:
560  // db->CompactRange(options, nullptr, nullptr);
561  // Note that after the entire database is compacted, all data are pushed
562  // down to the last level containing any data. If the total data size after
563  // compaction is reduced, that level might not be appropriate for hosting all
564  // the files. In this case, client could set options.change_level to true, to
565  // move the files back to the minimum level capable of holding the data set
566  // or a given level (specified by non-negative options.target_level).
567  virtual Status CompactRange(const CompactRangeOptions& options,
