53  // If a transaction has a snapshot set, the transaction will ensure that
54  // any keys successfully written(or fetched via GetForUpdate()) have not
55  // been modified outside of this transaction since the time the snapshot was
56  // set.
57  // If a snapshot has not been set, the transaction guarantees that keys have
58  // not been modified since the time each key was first written (or fetched via
59  // GetForUpdate()).
60  //
61  // Using SetSnapshot() will provide stricter isolation guarantees at the
62  // expense of potentially more transaction failures due to conflicts with
63  // other writes.
64  //
65  // Calling SetSnapshot() has no effect on keys written before this function
66  // has been called.
67  //
68  // SetSnapshot() may be called multiple times if you would like to change
69  // the snapshot used for different operations in this transaction.
70  //
71  // Calling SetSnapshot will not affect the version of Data returned by Get()
72  // methods. See Transaction::Get() for more details.
73  virtual void SetSnapshot() = 0;
74 
75  // Similar to SetSnapshot(), but will not change the current snapshot
76  // until Put/Merge/Delete/GetForUpdate/MultigetForUpdate is called.
77  // By calling this function, the transaction will essentially call
78  // SetSnapshot() for you right before performing the next write/GetForUpdate.
79  //
80  // Calling SetSnapshotOnNextOperation() will not affect what snapshot is
81  // returned by GetSnapshot() until the next write/GetForUpdate is executed.
82  //
83  // When the snapshot is created the notifier's SnapshotCreated method will
84  // be called so that the caller can get access to the snapshot.
85  //
86  // This is an optimization to reduce the likelihood of conflicts that
87  // could occur in between the time SetSnapshot() is called and the first
88  // write/GetForUpdate operation. Eg, this prevents the following
89  // race-condition:
90  //
91  // txn1->SetSnapshot();
92  // txn2->Put("A", ...);
93  // txn2->Commit();
94  // txn1->GetForUpdate(opts, "A", ...); // FAIL!
95  virtual void SetSnapshotOnNextOperation(
96  std::shared_ptr<TransactionNotifier> notifier = nullptr) = 0;
